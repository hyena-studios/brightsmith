## Decisions Made
1. **Semver classification wraps existing diff rather than replacing it.** The current `contract.diff_contract()` returns BREAKING/NON_BREAKING. The CAB module refines that binary into PATCH/MINOR/MAJOR. This avoids rewriting contract.py and keeps the CAB logic self-contained.
2. **MAJOR changes always require human approval regardless of REQUIRE_HUMAN_APPROVAL toggle.** Breaking schema changes are too consequential to auto-approve. This is the one exception to the global toggle.
3. **Pipeline position: after primary-agent, before governance-reviewer-post.** The agent needs the implementation to exist (to diff against the contract), but must fire before governance review so that governance-reviewer can verify the CAB decision was made.
4. **cab-review step is skippable.** For new tables (no existing contract), the step is skipped with documented justification. This avoids blocking greenfield specs.
5. **Gold step tuples alias Silver.** Since `GOLD_GREENFIELD_STEPS = SILVER_GREENFIELD_STEPS`, only Silver tuples need modification. This is an existing pattern in the codebase.
6. **JSON schemas are the contract between Brightsmith and Brightforge.** The governance artifact schemas defined in this spec are what Brightforge Spec 18 reads directly. Getting them right means the UI layer builds cleanly on top.
7. **dq-engineer requires cab-review in greenfield.** This ensures the CAB decision is made before DQ execution validates the new schema. In backfill, cab-review runs after dq-engineer since the schema already exists.
8. **governance-reviewer-post requires cab-review.** This lets governance-reviewer verify that a CAB decision exists for specs that modify existing tables.
